[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture to see how it works under the hood. 
First, every user query starts at the START node and enters the Security Checkpoint node. This checkpoint acts as a secure gateway, automatically sanitizing HTML, redacting PII, and checking for prompt injection signatures.
If a prompt injection is detected, it routes immediately to the Security Alert node, blocking execution and logging the attempt.
If the input is clear, it routes to the orchestrator agent, our central coordinator.
The orchestrator agent delegates specialized analysis tasks to the analyzer_agent, which uses our local Model Context Protocol server tools to query firewall logs, look up Indicator of Compromise reputation scores, and retrieve CVE details.
The orchestrator then delegates the findings to the remediator_agent to compile containment recommendations and generate a structured markdown incident report.
Finally, the flow moves to the human_review node. Here, the workflow graph pauses using the ADK RequestInput mechanism.
Only when a human administrator explicitly approves the action does the graph transition to the execute_remediation node to block the threat. If denied, it transitions to the cancel_remediation node.

[3:00 - SECURITY & MCP]
(show assets/architecture_diagram.png)
What makes this system exceptionally secure is our two-layer defense. First, the Security Checkpoint programmatically scrubs emails, bearer tokens, and credentials, meaning sensitive data never leaves our network boundary or enters the model's training context. Second, our custom Model Context Protocol server keeps all tool executions decoupled from the core LLM orchestration.
